# CS Core

> A deep exploration of computer science from first principles — from execution to systems to distributed computing.

---

##  What this is

CS Core is a structured notebook for understanding how computation actually works.

It focuses on:

- how programs execute on hardware
- how operating systems manage resources
- how abstractions are built and broken
- how distributed systems coordinate state

---

## Core Layers of Computer Science

### Execution Layer
How code becomes execution on a machine:

- CPU instruction cycles
- processes and threads
- memory layout (stack, heap, registers)
- system calls
- context switching

---

### Operating Systems
How machines manage computation:

- scheduling
- virtual memory
- isolation and protection
- filesystems
- resource management

---

### Runtime Systems
How languages actually run:

- compilers vs interpreters
- virtual machines
- garbage collection
- JIT compilation
- language execution models

---

### Distributed Systems
How computation scales beyond one machine:

- consensus and coordination
- replication and fault tolerance
- distributed state
- networking fundamentals
- consistency models

---

## Philosophy

> “Every abstraction should be explainable by the layer below it.”

CS Core is built on three principles:

- **First principles thinking** — no blind abstractions
- **Execution awareness** — understand what actually runs
- **Layered reasoning** — move from hardware upward

---

## Learning Method

Every topic is explored through:

- mental models
- diagrams of execution flow
- real code experiments
- system-level reasoning
